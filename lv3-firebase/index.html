互動式能力階梯測驗 - Lv.3

即時競賽版・作答完分數自動同步大螢幕排行榜

1. 下列關於 Lv.1 純前端（算完就忘）架構的敘述，何者正確？

A. 資料會存存放於 Google 試算表中
B. 關閉網頁或重新整理後，先前的作答資料就會消失
C. 必須撰寫複雜的 Firebase 安全規則
D. 可以安全地隱藏 AI API Key
解析：B 是正確答案。Lv.1 完全運行在瀏覽器記憶體中，關閉或重新整理網頁後資料即消失，無任何資料庫或後端，因此零維護成本且隱私極度安全。

2. 老師想以「非即時」方式收集學生成績，且希望在自己最熟悉的試算表介面查看數據，最適合哪一級？

A. Lv.1 純前端
B. Lv.2 Google Sheets 後端
C. Lv.3 Firebase 後端
D. 前後端 + AI
解析：B 是正確答案。Lv.2 使用 Google Sheets 作為後端，資料直接寫入試算表，方便老師讀取、排序和分析，非常適合非即時的成績登錄。

3. 關於 Lv.3 Firebase 後端（即時競賽版）與 Lv.2 Google Sheets 後端（課後回收版）的主要差別，下列敘述何者正確？

A. Lv.3 可以做到「秒級」的多人即時同步與大螢幕排行榜更新
B. Lv.3 寫入速度較慢，比較適合非即時的課後回饋
C. Lv.2 不需要透過任何後端代理就能把資料直接存入
D. Lv.2 可以承載比 Lv.3 更高頻率的多人同時寫入
解析：A 是正確答案。Lv.3 使用 Firebase 即時資料庫，具備極強的即時同步能力，非常適合課堂當下的多人搶答與即時排行榜；而 Lv.2 (Google Sheets) 寫入與響應速度較慢，不適合秒級同步。

4. 在 Lv.4 中，為了不讓 Groq API Key 在前端瀏覽器中外洩（被按 F12 檢視原始碼偷走），正確的做法是？

A. 將 Key 寫在前端 JavaScript 的變數中，並進行加密
B. 將 Key 放在 Firebase Firestore 的公開欄位
C. 使用 Netlify Functions 當作後端代理，並將 Key 存放於伺服器環境變數中
D. 將 Key 存放於 Google 試算表內，由前端直接 Fetch 讀取
解析：C 是正確答案。Netlify Functions 在伺服器端運行，能讀取環境變數，前端只能呼叫其端點而碰不到金鑰本身，非常安全。
